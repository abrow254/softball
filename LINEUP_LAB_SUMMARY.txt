## SECTION 2: DATA LAYER

File: lib/db/lineupLab.ts

Function: getLineupLabData(seasonId: string) → Promise<LineupLabPlayer[]>

Flow:
1. Query season_stats for rate stats (avg, obp, slg, ops) + is_regular flag
2. Query player_game_log for all (player, game_date, ops) in season
3. Query players table for gender (M | F | null) for each player in season_stats
4. For each player:
   - Extract last 6 game OPS values (oldest-first, for sparkline)
   - Compute mean of last 3 games as recentForm (hot/cold indicator)
5. Return LineupLabPlayer[] (typed interface with all fields)

Form window: last 6 games; recent-form metric: mean of last 3 games.

## SECTION 3: OPTIMIZER ALGORITHM

File: lib/optimizer.ts

### Data Model

SlotConfig = { role: string, w: number (importance), a: number (OBP%), b: number (SLG%) }

Slot weights (slots 1–8, spec-exact):
  1. Leadoff:  w=1.00, a=.65, b=.35
  2. 2-hole:   w=1.05, a=.58, b=.42  ← highest weight
  3. 3-hole:   w=.80,  a=.50, b=.50
  4. Cleanup:  w=1.00, a=.40, b=.60  ← second-highest weight
  5. 5-hole:   w=.90,  a=.45, b=.55
  6. 6-hole:   w=.74,  a=.55, b=.45
  7. 7-hole:   w=.66,  a=.50, b=.50
  8. 8-hole:   w=.60,  a=.50, b=.50

Extended slots (9+): descending weights (0.55, 0.50, 0.46, 0.42, ...) with a=b=0.50

### Slot value formula

value(player, slot) = w × (a × obp + b × slg)
lineupScore = Σ value(order[i], slot[i]) for i in 0..n-1

### MMF constraint (cyclic)

No 3 consecutive males in the batting order, with the wrap from last batter to first counting.

isCyclicViolation(i) iff:
  order[i].gender == 'M' AND
  order[(i+1) % n].gender == 'M' AND
  order[(i+2) % n].gender == 'M'

Feasibility guard: a legal order exists iff males ≤ 2 × nonMales (F + null)

### Algorithm selector

if n <= 8:
  return bruteForce(players, slots)
else:
  return greedyOptimize(players, slots)

### Brute force (n ≤ 8)

- Generate all n! permutations via Heap algorithm (recursive swap)
- For each permutation, check MMFViolations()
- Keep the permutation with the highest lineupScore among legal ones
- Max iterations: 8! = 40,320

### Greedy pattern-based (n > 8)

1. Enumerate all valid cyclic gender patterns:
   - Pattern = array of length n, each element 0 (non-male) or 1 (male)
   - Exactly `males` 1s and `nonMales` 0s
   - No 3 consecutive 1s (linear pruning during generation)
   - Final check: cyclic wrap is also valid (no wrap-around 3-in-a-row)

2. For each valid pattern:
   - Identify male slots (pattern[i] == 1) and non-male slots (pattern[i] == 0)
   - Sort slots by w descending (most valuable first) within each group
   - Sort players by OPS descending within each gender
   - Assign greedily: i-th best player of gender G → i-th most-valuable slot for G
   - Compute lineupScore for this assignment

3. Return assignment with highest lineupScore across all patterns

## SECTION 4: UI COMPONENTS

File: components/LineupLab.tsx

Mobile-first design, Cardinals red + mustard accent, Oswald headings.

### Lineup card (hero section)

For each batting-order slot (1–n):
  - Slot number (1-indexed, red text)
  - Role label ("Leadoff", "2-hole", ..., or "N-hole" for n > 8)
  - Player name (truncated on mobile)
  - Gender marker (M/F/? badge, colors: blue for M, pink for F, gray for null)
  - Form badge (Hot ▲ / Cold ▼ / Steady ~, colors: orange / sky / gray)
  - Action buttons: ▲ ▼ (nudge up/down) + ✕ (send to bench)
  - Stats row (AVG OBP SLG OPS in tabular-nums mono font)
  - Sparkline (6-point line chart, green/red/gray based on trend)
  - Left rail: RED on MMF-violated slots, transparent otherwise

### Status chips (top)

- Legality: ✓ Legal | ✕ Illegal MMF | ✕ No legal order
- Lineup score (tabular mono)
- Action buttons: Auto-optimize (disabled if infeasible) + Reset

### Infeasibility warning

If males > 2 × females: banner explaining the constraint and suggesting remedies.

### Selection banner

During bench selection: "Tap a slot to swap [NAME] in, or tap the player again to cancel."

### Bench section

For each bench player (if any):
  - Same format as lineup cards but more compact
  - Tap to select (gold ring highlight + "selected" state)
  - "+ Add" button to add directly to order
  - During selection: note "Tap a slot in the order to swap [NAME] in"

### The Read (insight panel)

Plain-language sentences generated from order + violations + form:
  - Always: "N leads off at X OBP. M hits second (Y OPS). C cleans up, Z SLG."
  - If heating up (last-3 > season OPS + 0.04): "[Player] is heating up — last-3 OPS X vs Y season."
  - If cooling off (season OPS - last-3 > 0.04): "[Player] cooled off — consider dropping a spot."
  - If infeasible: "No legal order — add a woman or take an auto-out."
  - If violation: "Illegal: slots [N, N+1, N+2] run three men in a row. Move a woman between them."

### Slot-value explainer bar chart

Horizontal bar chart showing slots 1–8 with relative widths proportional to w.
Slots 2 and 4 highlighted (gold).
Caption: "Slots 2 and 4 carry the most weight — put your best bats there."

### Interaction model (mobile-first, no drag-drop)

- Tap a bench player to select (toggle; gold ring + status banner)
- While a bench player is selected, tap a lineup slot to swap them in
- Tap ▲/▼ on a lineup slot to nudge the player up/down
- Tap ✕ on a lineup slot to send the player to bench
- Auto-optimize button: runs optimize(), animates result, shows 🦜 cockatiel flash
- Reset button: returns to initial state (top 8 by OPS in order, rest on bench)

### Auto-optimize animation

On success:
  - Order state updates
  - Flash message: "🦜 Order optimized — [score] lineup score."
  - Lasts 1.8 seconds, then fades

## SECTION 7: HOT/COLD LOGIC

File: components/LineupLab.tsx, function getFormTag()

Thresholds:
  - Hot: recentForm > seasonOPS × 1.06
  - Cold: recentForm < seasonOPS × 0.94
  - Steady: otherwise

recentForm = mean OPS of last 3 games (0 if fewer than 1 game)
seasonOPS = season stat line OPS

Display:
  - Hot: ▲ orange badge
  - Cold: ▼ sky badge
  - Steady: ~ gray badge

## SECTION 8: HOUSE FORMULAS (UNCHANGED)

Formulas preserved from Google Sheet via formulas.ts:

  hits = singles + doubles + triples + hr
  tb = singles + 2×doubles + 3×triples + 4×hr
  pa = runs + hits + bb + hbp + roe  [HOUSE RULE]
  avg = hits / ab
  obp = (hits + fc) / ab  [HOUSE RULE: counts FC, divides by AB not PA]
  slg = tb / ab
  ops = avg + slg  [HOUSE RULE: NOT obp + slg]
  iso = slg - avg
  xbh_pct = (doubles + triples + hr) / hits

All applied identically in SQL view and TypeScript.
